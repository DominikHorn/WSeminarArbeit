                                                                                  Erik Michael James Williams a year ago

                                                                                  You should change: "return r < (a.x + b.x)^2 + (a.y + b.x)^2", in CirclevsCircleOptimized, one of the first code blocks. It should be:
                                                                                  "return r > (a.x - b.x)^2 + (a.y - b.y)^2".                                                                                                                                 RandyGaul CrazyMadMax a year ago

                                                                                                                                Hi! Good question. The problem you are observing is that each solution (impulse) is calculated and applied in isolation of all other solutions. The solution is to solve for each impulse like a linear algebra problem of solving a system of equations.

                                                                                                                                Currently the most popular approach is to implemented Projected Gauss Seidel, an iterative LCP solver. The actual implementation you seek comes down to storing impulses from frame to frame, and clamping the accumulated result above zero. This will let negative impulses by applied when necessary, and helps to drive convergence over the course of multiple frames.

                                                                                                                                This topic is a bit involved so don't feel bad if it's difficult to implement at first! There's a lot of information to learn.
